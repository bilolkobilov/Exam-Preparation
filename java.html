Module I

1. Concepts of OOP (Object-Oriented Programming)

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects," which can contain data and code:

  • Encapsulation: Bundling data (attributes) and methods that operate on the data into a single unit (class). It helps in data hiding.
  • Abstraction: Hiding complex implementation details and showing only the necessary features of an object.
  • Inheritance: A mechanism where a new class (subclass/derived class) inherits properties and behaviors from an existing class (superclass/base class).
  • Polymorphism: The ability of a variable, function, or object to take on multiple forms.

Advantages of OOP over procedural programming:

  • Simpler software development and maintenance
  • Improved code reusability
  • Better data protection
  • Ability to model real-world objects more effectively

Java code examples:


// Encapsulation
class Circle {
    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    public double getArea() {
        return Math.PI * radius * radius;
    }
}

// Abstraction
abstract class Shape {
    abstract double getArea();
}

class Rectangle extends Shape {
    double width;
    double height;

    public Rectangle(double width, double height) {
        this.width = width;
        this.height = height;
    }

    @Override
    double getArea() {
        return width * height;
    }
}

// Inheritance
class Vehicle {
    String modelName;
    void start() {
        System.out.println("Vehicle started");
    }
}

class Car extends Vehicle{
    String carName;
}

// Polymorphism
class Animal {
    void makeSound() {
        System.out.println("Generic sound");
    }
}

class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("Woof");
    }
}

public class Main {
    public static void main(String[] args) {
        Circle circle = new Circle(5);
        System.out.println("Circle Area: " + circle.getArea());

        Rectangle rectangle = new Rectangle(4, 6);
        System.out.println("Rectangle Area: " + rectangle.getArea());

        Animal myDog = new Dog();
        myDog.makeSound(); // Calls the Dog's makeSound()
    }
}

2. Features of Java

Key features of Java:

  • Platform Independence: Java code can run on any platform with a JVM. The Java Virtual Machine (JVM) is responsible for converting Java bytecode into machine code.
  • Object-Oriented: Java is an OOP language.
  • Multithreading: Supports concurrent execution of multiple threads.
  • Memory Management (Garbage Collection): Automatic memory management.
  • Exception Handling: Provides a mechanism to handle runtime errors.

3. How Java is Different from C++

Comparison of Java and C++:

  • Memory Management: Java has automatic garbage collection; C++ requires manual memory management.
  • Syntax: Java's syntax is simpler and more consistent.
  • Performance: C++ is generally faster due to direct hardware access.
  • Pointers: Java does not have explicit pointers. C++ uses pointers.
  • Multiple Inheritance: Java doesn't support multiple inheritance of classes (but supports it through interfaces); C++ supports it.
  • Exception Handling: Java has built-in exception handling; C++ has exception handling, but it's not as enforced.

4. Data Types & Control Statements

Java Data Types:

  • Primitive Data Types: `byte`, `short`, `int`, `long`, `float`, `double`, `boolean`, `char`.
  • Non-Primitive Data Types: Classes, Objects, Arrays, Strings, Interfaces.

Control Statements:

  • Conditional Statements: `if`, `else if`, `else`, `switch`.
  • Loops: `for`, `while`, `do-while`.

// Example of control statements
public class ControlStatements {
    public static void main(String[] args) {
        int x = 10;

        if (x > 15) {
            System.out.println("x is greater than 15");
        } else if (x > 10) {
            System.out.println("x is greater than 10");
        } else {
            System.out.println("x is not greater than 10");
        }

        for (int i = 0; i < 5; i++) {
            System.out.println("Iteration: " + i);
        }

        int y = 0;
        while (y < 5) {
            System.out.println("While loop: " + y);
            y++;
        }

        int z = 0;
        do {
            System.out.println("Do-while loop: " + z);
            z++;
        } while (z < 5);
    }
}

5. Identifiers, Arrays, Operators, Classes, and Constructors

Identifiers:

A name given to a class, method, variable, or other user-defined item in a program. Rules for defining identifiers:

  • They should not begin with a digit
  • They can contain letters, digits, underscores, and dollar signs
  • They are case-sensitive

Operators:

  • Arithmetic Operators: `+`, `-`, `*`, `/`, `%`.
  • Relational Operators: `==`, `!=`, `>`, `<`, `>=`, `<=`.
  • Logical Operators: `&&`, `||`, `!`.
  • Bitwise Operators: `&`, `|`, `^`, `~`, `<<`, `>>`, `>>>`.

Arrays:

A collection of similar data type elements stored at contiguous memory locations.

  • One-dimensional Array: `int[] arr = new int[5];`
  • Multi-dimensional Array: `int[][] arr = new int[3][3];`

Classes:

A blueprint or template for creating objects.

Constructors:

Special methods used to initialize objects.

  • Default Constructor: A constructor with no parameters.
  • Parameterized Constructor: A constructor with parameters.

class MyClass {
    int x;
    int y;

    // Default Constructor
    MyClass() {
        x = 0;
        y = 0;
    }

    // Parameterized Constructor
    MyClass(int a, int b) {
        x = a;
        y = b;
    }
}

public class Main {
    public static void main(String[] args) {
        int[] myArray = {1, 2, 3, 4, 5};
        System.out.println(myArray[0]);

        MyClass obj1 = new MyClass();  // Calls default constructor
        MyClass obj2 = new MyClass(10, 20); // Calls parameterized constructor
    }
}

6. Inheritance: Multilevel Hierarchy

Inheritance is a mechanism where a new class (subclass) inherits properties and behaviors from an existing class (superclass).

  • Single Inheritance: A class inherits from only one superclass.
  • Multilevel Inheritance: A class inherits from a subclass, which in turn inherits from another class.
  • Hierarchical Inheritance: Multiple classes inherit from a single superclass.

The `super` keyword is used to call the constructor or method of the superclass. Method overriding is when a subclass provides a different implementation of a method defined in its superclass. 


class GrandParent {
    String grandParentName = "GrandParent";
}

class Parent extends GrandParent {
    String parentName = "Parent";
}

class Child extends Parent {
    String childName = "Child";

    public void displayDetails() {
        System.out.println("GrandParent Name: " + grandParentName);
        System.out.println("Parent Name: " + parentName);
        System.out.println("Child Name: " + childName);
    }
}

public class Main {
    public static void main(String[] args) {
        Child child = new Child();
        child.displayDetails();
    }
}

7. Method Overriding, Abstract Classes, and Final Classes

Method Overriding:

When a subclass provides a different implementation of a method defined in its superclass. It enables runtime polymorphism.

Abstract Classes:

A class that cannot be instantiated. It may contain abstract methods (methods without implementation).

Final Classes:

A class that cannot be subclassed. The `final` keyword can also be used with methods to prevent them from being overridden. 


abstract class MyAbstractClass {
    abstract void myMethod(); // Abstract method

    void myNonAbstractMethod() {
        System.out.println("Non-abstract method");
    }
}

class MySubclass extends MyAbstractClass {
    @Override
    void myMethod() {
        System.out.println("Implementation of myMethod in subclass");
    }
}

final class MyFinalClass {
    // This class cannot be subclassed
}

class MyClass {
    final void myFinalMethod() {
       //This method cannot be overridden
    }
}

public class Main {
    public static void main(String[] args) {
       MySubclass obj = new MySubclass();
       obj.myMethod();
    }
}

8. String Class & Various Types of String Operations

In Java, strings are immutable objects. The `String` class is used to create and manipulate strings.

`StringBuilder` and `StringBuffer` are used for mutable strings. `StringBuilder` is faster but not thread-safe, while `StringBuffer` is thread-safe.

String Operations:

  • Concatenation
  • Comparison
  • Substring extraction
  • Searching
  • Replacing

public class StringOperations {
    public static void main(String[] args) {
        String str1 = "Hello";
        String str2 = "World";

        // Concatenation
        String result = str1 + " " + str2;
        System.out.println("Concatenation: " + result);

        // Comparison
        boolean isEqual = str1.equals("Hello");
        System.out.println("Comparison: " + isEqual);

        // Substring extraction
        String sub = str2.substring(2, 5);
        System.out.println("Substring: " + sub);

        // Searching
        int index = str1.indexOf("l");
        System.out.println("Index of 'l': " + index);

        // Replacing
        String replaced = str1.replace("l", "p");
        System.out.println("Replaced string: " + replaced);

        //StringBuilder
        StringBuilder sb = new StringBuilder("Hello");
        sb.append(" World");
        System.out.println("StringBuilder: " + sb.toString());

        //StringBuffer
        StringBuffer sbuf = new StringBuffer("Hello");
        sbuf.append(" World");
        System.out.println("StringBuffer: " + sbuf.toString());
    }
}

9. Runtime Polymorphism

Runtime polymorphism, also known as dynamic polymorphism, is achieved through method overriding. The actual method to be called is determined at runtime. 


class Superclass {
    void display() {
        System.out.println("Superclass display method");
    }
}

class Subclass extends Superclass {
    @Override
    void display() {
        System.out.println("Subclass display method");
    }
}

public class Main {
    public static void main(String[] args) {
        Superclass obj = new Subclass(); // Upcasting
        obj.display(); // Calls Subclass's display() at runtime
    }
}

Module II

10. Defining, Implementing, and Importing Packages

A package is a mechanism for organizing Java classes into namespaces. Packages help in managing large codebases and preventing naming conflicts.

Defining a package:

Use the `package` keyword at the beginning of a Java file.

Importing a package:

Use the `import` keyword to use classes from a package. 


// Defining a package: com.example.mypackage
package com.example.mypackage;

public class MyClass {
    public void myMethod() {
        System.out.println("MyClass from com.example.mypackage");
    }
}

// Another file
// Importing and using the package
import com.example.mypackage.MyClass;

public class Main {
    public static void main(String[] args) {
        MyClass obj = new MyClass();
        obj.myMethod();
    }
}

11. Interfaces: Functional Interface and Lambda Expression

An interface is a collection of abstract methods. It is used to achieve abstraction and multiple inheritance in Java.

Functional Interface:

An interface with a single abstract method.

Lambda Expression:

A concise way to represent an anonymous function. It can be used to implement functional interfaces. 


// Functional Interface
interface MyInterface {
    void myMethod();
}

public class Main {
    public static void main(String[] args) {
        // Lambda Expression
        MyInterface obj = () -> System.out.println("Lambda expression implementation");
        obj.myMethod();
    }
}

12. Errors and Exception Handling

Errors indicate serious problems that a reasonable application should not try to handle. Exceptions are conditions that an application might want to handle.

Types of Exceptions:

  • Checked Exceptions: Exceptions that are checked at compile time. The compiler forces you to handle them.
  • Unchecked Exceptions: Exceptions that are not checked at compile time.

Exception Handling Keywords:

  • `try`: Encloses the code that might throw an exception.
  • `catch`: Handles the exception.
  • `finally`: A block that is always executed (after `try` and `catch` blocks).
  • `throw`: Used to explicitly throw an exception.
  • `throws`: Used in a method signature to declare that a method might throw an exception.

public class ExceptionHandling {
    public static void main(String[] args) {
        try {
            int result = 10 / 0; // This will throw an ArithmeticException
            System.out.println("Result: " + result); // This line will not be executed
        } catch (ArithmeticException e) {
            System.out.println("Caught ArithmeticException: " + e.getMessage());
        } finally {
            System.out.println("Finally block executed");
        }
        System.out.println("Program continues...");
    }
}

13. Java’s Built-in Exceptions and Custom Exceptions

Java provides many built-in exceptions, such as:

  • `NullPointerException`
  • `ArrayIndexOutOfBoundsException`
  • `IOException`
  • `ClassNotFoundException`

Custom Exceptions:

You can create your own exception classes by extending the `Exception` class or its subclasses. 


// Custom Exception
class MyException extends Exception {
    public MyException(String message) {
        super(message);
    }
}

public class CustomException {
    public static void main(String[] args) {
        try {
            validateAge(15);
        } catch (MyException e) {
            System.out.println("Caught MyException: " + e.getMessage());
        }
    }

    static void validateAge(int age) throws MyException {
        if (age < 18) {
            throw new MyException("Age must be 18 or older");
        } else {
            System.out.println("Age is valid");
        }
    }
}

14. Exception Logging

Logging exceptions is crucial for debugging and monitoring applications. Java provides the `java.util.logging` package. Log4j is another popular logging framework. 


import java.util.logging.Logger;
import java.util.logging.Level;

public class LoggingExample {
    private static final Logger LOGGER = Logger.getLogger(LoggingExample.class.getName());

    public static void main(String[] args) {
        try {
            int result = 10 / 0;
        } catch (ArithmeticException e) {
            LOGGER.log(Level.SEVERE, "ArithmeticException occurred", e);
        }
    }
}
