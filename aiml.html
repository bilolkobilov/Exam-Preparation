1. Defining Artificial Intelligence

Artificial Intelligence (AI) is a field of study that enables machines to simulate human intelligence. It involves the development of computer systems capable of performing tasks that typically require human intelligence, such as learning, problem-solving, decision-making, and language understanding.

Real-world examples of AI include:

  • Self-driving cars (e.g., Tesla Autopilot)
  • Chatbots (e.g., ChatGPT, Replika)
  • Medical diagnosis systems
  • Recommendation systems (e.g., Netflix, Amazon)

There are two main types of AI:

  • Strong AI (or Artificial General Intelligence - AGI): A hypothetical form of AI that possesses human-level cognitive abilities. It can understand, learn, and apply knowledge across a wide range of tasks, much like a human. Strong AI does not currently exist.
  • Weak AI (or Narrow AI): AI that is designed and trained to perform a specific task. Most AI systems today, such as virtual assistants (Siri, Alexa), recommendation systems, and image recognition software, are examples of weak AI.

3. Using Predicate Logic and Representing Knowledge as Rules

First-order logic (FOL), also known as predicate logic, is a formal system used in AI to represent knowledge. It extends propositional logic by allowing variables, quantifiers (∀ for all, ∃ for some), and predicates.

Example of FOL:

  • ∀x (Human(x) → Mortal(x)) (For all x, if x is a human, then x is mortal)
  • ∃x (Student(x) ∧ Smart(x)) (There exists an x such that x is a student and x is smart)

Prolog is a logic programming language often used to implement AI systems based on predicate logic.

Prolog example: 


parent(john, mary).  % john is a parent of mary.
parent(john, david).
parent(jane, mary).
parent(jane, david).
grandparent(X, Y) :- parent(X, Z), parent(Z, Y). % Rule for grandparent

In this Prolog example, `parent(john, mary)` is a fact, and `grandparent(X, Y) :- parent(X, Z), parent(Z, Y)` is a rule. The rule states that X is a grandparent of Y if X is a parent of Z, and Z is a parent of Y.

4. Representing Simple Facts in Logic

Knowledge representation in logic involves expressing facts and rules in a formal language that a computer can understand and process. Facts are basic truths about the world.

Examples of representing simple facts:

  • Fact: "The sky is blue." Representation in Prolog: `sky_color(blue).`
  • Fact: "John is a student." Representation in FOL: `Student(john).`
  • Fact: "Mary is a doctor and works at General Hospital." Representation in FOL: `Doctor(mary) ∧ WorksAt(mary, general_hospital).`

Rules define relationships between facts or how new facts can be derived from existing ones.

5. Computable Functions and Predicates

In the context of AI and logic, computable functions and predicates are fundamental concepts.

  • Computable Function: A function that can be evaluated by a Turing machine or, equivalently, by an algorithm. A function is computable if there exists an algorithm that can take any valid input and produce the corresponding output in a finite number of steps.
  • Predicate: A function that returns a truth value (true or false). In logic, predicates are used to express properties of objects or relationships between them.

Turing Computability: The concept of Turing computability is central to understanding the limits of what can be computed by machines. The Church-Turing thesis states that any function that is "naturally regarded as computable" can be computed by a Turing machine.

Example of a computable function in Python: 


def add(x, y):
    """
    This function takes two numbers as input and returns their sum.
    It is a computable function because it can be implemented as an algorithm.
    """
    return x + y

result = add(5, 3)  # result will be 8

6. Procedural vs Declarative Knowledge

In AI, knowledge can be represented in two main ways: procedural and declarative.

  • Procedural Knowledge: Knowledge about how to do things. It is represented as a set of instructions or a sequence of steps for solving a problem. It focuses on the process of achieving a result.
  • Declarative Knowledge: Knowledge about what is true. It is represented as a set of facts and relationships, without specifying how to use them. It focuses on describing the problem or the domain.

Examples:

  • Procedural:
    • A recipe for baking a cake (sequence of steps)
    • A computer program that calculates the factorial of a number (algorithm)
  • Declarative:
    • The statement "The capital of France is Paris."
    • A database containing facts about customers and their orders.

7. Logic Programming

Logic programming is a programming paradigm based on formal logic. It uses logical statements to represent knowledge and rules, and uses a logical inference engine to derive conclusions.

Prolog: Prolog is a prominent logic programming language. It is used for AI applications such as:

  • Knowledge representation and reasoning
  • Expert systems
  • Natural language processing
  • Database querying

Sample Prolog program: 


% Facts:
male(john).
male(david).
female(mary).
female(jane).
parent(john, mary).  % john is a parent of mary
parent(john, david).
parent(jane, mary).
parent(jane, david).

% Rules:
father(X, Y) :- male(X), parent(X, Y).
mother(X, Y) :- female(X), parent(X, Y).
sibling(X, Y) :- parent(Z, X), parent(Z, Y), X \= Y. % X and Y are siblings if they have the same parent and are not the same person.

In this program:

  • `male(john).`, `female(mary).`, etc., are facts.
  • `father(X, Y) :- male(X), parent(X, Y).` is a rule. It states that X is the father of Y if X is male and X is a parent of Y.
  • `\`= is the "not equals" operator.

8. Mathematical Foundations: Matrix Theory and Statistics for ML

Matrix theory and statistics are essential mathematical foundations for machine learning.

  • Matrix Theory:
    • Matrix operations (addition, multiplication, transposition, inversion) are fundamental to many ML algorithms.
    • Matrices are used to represent data (e.g., feature matrices), linear transformations, and the weights in neural networks.
    • Eigenvalues and eigenvectors are used in dimensionality reduction techniques like Principal Component Analysis (PCA).
  • Statistics:
    • Descriptive statistics (mean, median, standard deviation) are used to understand and summarize data.
    • Probability theory is essential for understanding the uncertainty in data and model predictions.
    • Inferential statistics (hypothesis testing, confidence intervals) are used to draw conclusions about populations from sample data.
    • Distributions (normal, binomial, Poisson) are used to model data and make predictions.

Python code for matrix multiplication using NumPy:

9. Idea of Machines Learning from Data

Machine learning algorithms learn patterns and relationships from data without being explicitly programmed. They use historical data (training data) to build a model, which can then be used to make predictions or decisions on new, unseen data.

Machine Learning Pipeline:

  1. Data Collection: Gathering relevant data.
  2. Data Preprocessing: Cleaning, transforming, and preparing the data for training.
  3. Model Selection: Choosing an appropriate ML algorithm.
  4. Training: Training the model on the training data.
  5. Evaluation: Evaluating the model's performance on unseen data (test data).
  6. Deployment: Using the model to make predictions or decisions in the real world.
  7. Monitoring and Maintenance: Continuously monitoring the model's performance and retraining as needed.

10. Classification of Problems

Machine learning problems can be broadly classified into three categories:

  • Supervised Learning: The algorithm learns from labeled data, where the input features and the corresponding output labels are provided.
  • Unsupervised Learning: The algorithm learns from unlabeled data, where only the input features are provided, and the algorithm must discover patterns or structures in the data.
  • Reinforcement Learning: An agent learns to interact with an environment by performing actions and receiving rewards or penalties. The goal is to learn a policy that maximizes the cumulative reward.

from sklearn.linear_model import LinearRegression, LogisticRegression

# Sample data
# Regression data

13. Linear Regression: Model Representation for Single Variable

In linear regression with a single variable (also called simple linear regression), the goal is to find a linear relationship between the input variable (x) and the output variable (y).

The model is represented by the equation:

\( h_\theta(x) = \theta_0 + \theta_1x \)

Where:

  • \( h_\theta(x) \) is the predicted output (hypothesis) for a given input x.
  • \( x \) is the input variable.
  • \( \theta_0 \) is the y-intercept (the value of h when x is 0).
  • \( \theta_1 \) is the slope of the line (the change in h for a one-unit change in x).

Python implementation with data visualization: 


import numpy as np
import matplotlib.pyplot as plt
from sklearn.linear_model import LinearRegression

# Sample data
x = np.array([1, 2, 3, 4, 5]).reshape(-1, 1)  # Input variable (reshape for scikit-learn)
y = np.array([2, 4, 5, 4, 5])  # Output variable

# Create a Linear Regression model
model = LinearRegression()

# Train the model
model.fit(x, y)

# Get the parameters (theta_0 and theta_1)
theta_0 = model.intercept_
theta_1 = model.coef_[0]

print(f"Theta_0 (Intercept): {theta_0}")
print(f"Theta_1 (Slope): {theta_1}")

# Make predictions
x_test = np.array([1, 2, 3, 4, 5]).reshape(-1, 1)
y_pred = model.predict(x_test)

# Plot the data and the regression line
plt.scatter(x, y, color='blue', label='Actual data')
plt.plot(x_test, y_pred, color='red', label='Linear Regression')
plt.xlabel('x')
plt.ylabel('y')
plt.title('Linear Regression')
plt.legend()
plt.show()

14. Single Variable Cost Function

The cost function measures how well the linear regression model fits the training data. For single variable linear regression, the most common cost function is the Mean Squared Error (MSE).

The MSE is calculated as the average of the squared differences between the predicted values and the actual values.

Mathematical equation:

\( J(\theta) = \frac{1}{2m} \sum_{i=1}^{m} (h_\theta(x^{(i)}) - y^{(i)})^2 \)

Where:

  • \( J(\theta) \) is the cost function.
  • \( m \) is the number of training examples.
  • \( h_\theta(x^{(i)}) \) is the predicted output for the i-th input example \( x^{(i)} \).
  • \( y^{(i)} \) is the actual output for the i-th input example \( x^{(i)} \).
  • \( \theta \) represents the parameters of the model (\( \theta_0 \) and \( \theta_1 \) in simple linear regression).

Python implementation: 


import numpy as np

def compute_cost(x, y, theta):
    """
    Computes the Mean Squared Error (MSE) cost function for linear regression
    with one variable.

    Args:
        x: Input variable (numpy array of shape (m,)).
        y: Output variable (numpy array of shape (m,)).
        theta: Parameters of the model (numpy array of shape (2,)).
               theta[0] is the intercept, theta[1] is the slope.

    Returns:
        J: The cost (MSE) for the given data and parameters.
    """
    m = len(y)  # Number of training examples
    h = theta[0] + theta[1] * x  # Predicted values
    squared_errors = (h - y) ** 2
    J = (1 / (2 * m)) * np.sum(squared_errors)
    return J

# Sample data
x = np.array([1, 2, 3, 4, 5])
y = np.array([2, 4, 5, 4, 5])
theta = np.array([0, 1])  # Initial parameters (theta_0 = 0, theta_1 = 1)

# Compute the cost
cost = compute_cost(x, y, theta)
print(f"Cost with initial parameters: {cost}")

# Example with different parameters
theta = np.array([0.5, 1.5])
cost = compute_cost(x, y, theta)
print(f"Cost with theta = [0.5, 1.5]: {cost}")
